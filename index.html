Spring Bean Scopes

Posted on 2019-03-07

哇，有多久没有更新了。。。
想写个有趣的主题，把日常工作遇到的一些问题延伸带来的知识了解记录下来。
今日份：
Service里有一个cache loading的环节，在PST midnight的时候会重新refresh cache。因为cache里需要存多种数据，为了data consistency，会把所有的全部load到一个holder里面，等全部的data update执行完毕后再把这个holder release掉。
整个架构用的springboot。问题出现在scope上面。
Spring提供6种常见的scope，其中后四种只针对于web-aware的应用：

  • singleton
    这个顾名思义了，一个singleton的bean是指在container里面，只会有一个instance被创建出来。每次获取的instance都是同一个。如果一处改动了这个instance，其余的地方也会受到相应的变动。
    注： 如果scope没有manual define，那么默认的即为singleton

  • prototype
    这个和singleton相反，prototype的scope是指每次request获取都会拿到一个不一样的instance。

  • request
    对于网络应用来说，这个scope是指给每一个http request都新创建一个新的bean。

  • session, application, websocket (好。。这几个没咋用过。。。skip掉。。)

问题就出现在上面提到的注，我们其中的一个class忘记indicate prototype的scope，导致每次update cache其实都是在modify同一个map。虽说目前很幸运的并没有遇到什么问题。但是这相当于是新旧cache都在被用，data inconsistency。还好及时发现改掉了。感觉好多陷阱。。不小心忘记什么就GG了。。

Coffee Lover 2

Posted on 2018-02-06

上篇聊了聊自己和咖啡的恩怨情仇。这篇来说说自己学到的一些咖啡小知识。

几个常见的以及自己喜爱的咖啡。

  • Espresso 意式浓缩咖啡 -
    将极热（非沸腾）的水加压，约9⃣️个大气压，后冲煮研磨过的咖啡粉而制成。粉和水比例比较大，咖啡的苦涩味更重一些，故又叫浓缩咖啡。（没尝过）
  • Americano 美式咖啡 -
    简单说就是 2/5 Espresso + 3/5 水。没有掩盖咖啡本身味道的同时，又更清新一些，易让人接受。（凭啥比Espresso贵那么多）
  • Latte 拿铁 -
    十分milky的咖啡，大约 1/6 Espresso + 4/6 蒸汽牛奶 + 1/6 奶泡。将高温蒸汽打入牛奶，上层会形成奶泡，用勺子挡住奶泡，先将热蒸汽牛奶倒入杯中，再把丰富的奶泡添上去即可。据说所有奶质咖啡都要在Espresso做好的十秒之内就要倒入牛奶，否则会“shot 过期”。（家常饮品）
  • Cappuccino 卡布奇诺 -
    Foam更多一些的咖啡，1/3 Espresso + 1/3 蒸汽牛奶 + 1/3 奶泡。比起拿铁，口感更棉细一些。（其实如果不是两杯同时放我面前让我品尝，我是不会发现太多口感上的区分的 = =）这个名字的由来也是超级有意思。意大利人发现牛奶和咖啡柔软结合后形成的奶盖色泽，很像教会僧侣穿戴的棕色连帽长袍，就索性起了个和长袍一样的名字。（真性情）
  • Motca 摩卡 -
    巧克力注入的咖啡， 2/5 Espresso + 2/5 热巧克力 + 1/5 蒸汽牛奶。摩卡原本是指一种具有巧克力香气的咖啡豆，主要产于一个名叫摩卡的沿海小镇，所以就起名为摩卡豆。现今就直接在咖啡的基础上融入巧克力制造类似的口味了。听说摩卡里添加些酒精会别有一番风味。（想尝 0.0）
  • Macchiato 玛奇朵 -
    再Espresso的基础上加上两大勺绵密细软的奶泡即为一杯玛奇朵。这个名字也很有趣，意大利文里，Macchiato意为“烙印，印记”，好浪漫的词。约会时来一杯焦糖玛奇朵可就真的是“甜蜜的印记”了。（谁来约我来一杯）
  • Flat White 不知道中文是啥 -
    Espresso + Microfoam。对于拿铁来说，一般奶泡厚度会大于1cm，而Flat White一般不会超过5mm。卡布奇诺的粗奶泡一般是漂浮在牛奶上方的，而Flat White会更加细腻，质地很不同，并且牛奶和奶泡融合十分好，加上咖啡含量稍高，所以味道更加醇厚，口感更加顺滑。（西番这个）

最后来谈谈手冲咖啡。大致需要的有滤杯，底壶，滤纸和手冲壶这四样。整体的制作方法很简单，基本是按照1：15的粉水比，水温以90-93度为佳，先洗滤纸、再布粉、最后冲泡、滴滤、出杯。
制作黑咖啡还可以用法压壶等其他器具。
因为自己使用过法压壶，就来大致讲一下基本流程。先温壶，按自己喜好放入咖啡粉，然后加入一半约90度的烧水，搅拌少许，让咖啡油脂漂浮上来，再加入另一半烧水，盖盖子静置3-5分钟，缓慢的将滤器垂直向下压，即将咖啡渣压至壶底，最后倒入温好的咖啡杯中即可。
相比起浓郁醇厚的意式咖啡，手冲咖啡的高容错率，能够更好地体现咖啡豆本身的味道，并且没有其他添加物的影响，我们可以更直观的感受到咖啡豆本身特有的产地风味，比如随处可见的耶加雪菲，出来的手冲耶加，基本会完美呈现它本身的花香特质以及柑橘类风味！

来聊聊自己的咖啡趣事。
-1-
其实我并不是一个从一开始就很喜欢喝咖啡的人，最起初是因为想要提神，怀着新奇的心尝了尝，结果发现自己对咖啡因还挺敏感，心慌，发抖，睡不着，怕是得了啥病。而且感觉咖啡苦极了完全没办法理解那些‘每天一杯咖啡，兴奋自由一整天’的人。
-2-
后来估计是因为去了个巴厘岛，参观了当地的一个咖啡工厂，听了一番讲解，感觉有点儿意思（自己实在是个容易被洗脑的人啊orz），外加从那扛了好几包咖啡豆回家，又兴致满满买了个法压壶，看这架势东西这么齐乎，不品尝点也是不像样子。凑巧当时的室友是个咖啡控，每天早上来杯咖啡就成了惯例。上瘾这个东西可真不是瞎扯。慢慢发现自己对咖啡的敏感度没有那么高了，如果每日摄取量不算太多，自己完全可以handle。可那时候在法压壶滤出的咖啡里我还是会加入一些牛奶调和一下咖啡的苦涩，想着自己就这么踏入了咖啡的领域也是神奇，完美验证了‘生活要没有点形式感似乎也没法过下去’。
-3-
自己迷恋美式咖啡是从压力大并且功课多的大四开始的。自打开始了单人间生活，我的法压壶就再没露过脸，毕竟一个人的时候最懒。。。大四上学期每天又感觉没有咖啡提神简直过不下去，就开启了每天一杯美式的旅程。论咖啡因含量，其实美式和拿铁等都差不多，为啥我每天非喝美式呢？因为美式喝完我知道自己是喝了咖啡，其他的都给了我一种奶茶的幻觉 = = 可能这就叫自我意识影响情绪状态吧。不过美式真的可以很好的品尝到咖啡的香醇啊！可能有人说让我去喝Espresso，我拒绝。。。自我认知处于良好状态。。。逼格可能还没到那么高。。。
-4-
各种花式咖啡饮品翻牌过后意识到这么个小小的咖啡豆竟然和其他的味道融合的那么美好，而且每一种融合都展现独特的风味，甚是喜欢啊。想着了解多一些，所以一直很想去上个什么课来学点相关知识，好不容易看见星爸爸推送有个类似‘咖啡之旅’的活动，结果瞬间就没位置了 T_T 前两天喝着新学期第一杯拿铁，不想学习，心血来潮搜了各种样式的咖啡制作流程以及不同品种之间的区分。第二天再去星爸爸买咖啡的时候就特意的看了整个步骤，结果发现，喵的，根本不规范啊喂！我点的中杯卡布奇诺，那个Barista就淡定的给了我一个shot的espresso然后倒入了热牛奶，还没给我倒满，伐开森 = = 说好的奶泡呢喵的。。。而且Cappuccino难道不是1/3的咖啡么，喝完了感觉只有1/10的咖啡剩下全是牛奶，生气脸。一想，感觉自己之前喝咖啡都不会注意这些，无比生气脸。。。

Blog Setup :Day

Posted on 2017-12-22

The blog is built based on hexo, which is a fast, simple & powerful blog framework (as what their main page shows).

Step 1. Check dependencies

  • Node.js
  • Git

Step 2. Install Hexo

$ npm install hexo-cli -g
Use $ hexo -v to check the version

Step 3. Init Hexo

$ hexo init <folder>
If you wish to use GitHub Pages, like what I did, then:

  • Create a project for your GitHub Pages
    Simply go to your github website and create a new project named as <your-github-id>.github.io.
  • $ hexo init <your-githubt-id>.github.io

Once initialized, here’s what the projet will look like:
.
|– _config.yml
|– package.json
|– scaffolds
|– source
| |– _drafts
| |– _posts
|– themes

_config.yml

Site configuration file that contains most settings.

package.json

Application data. The EJS, Stylus and Markdown renders are installed by default. If you want, you can uninstall them later.

scaffolds

Scaffold folder. When a new post is created, Hexo will build files based on the corresponding file in scffolds folder. For example, when you run the command $ hexo new photo "Fantastic Pic", Hexo will try to find photo.md in the scaffolds folder and build the post based on it.

source

Source folder. This is where you put your site’s content. Hexo ignores hidden files and files or folder whose names are prefixed with _(underscore), except the _post folder. Renderable files, such as HTML, Markdown, will be processed and put into the public folder, while other files will simply be copied.

theme

Theme folder. Hexo generates a static website by combining the site content with the theme. The default one would be ‘landscape’

Step 4. Set Blog Information

Fill in and modify the _config.yml file.
Please find the link here, which explaines all the setting specified in the file.

Step 5. Deployment Settings

A detailed discription for deployment could be found here.
I use Git for deplyment and here is what I did.

  • Install hexo-deployer-git via command $ npm install hexo-deplyer-git --save
  • Edit deploy settings in _config.yml.

Step 6. Run Test Server

Now a basic website is complete. Run a test server for your blog by $ hexo server. Visit localhost:4000 to see your pages.

Step 7. Make a New Post

Make a new post by simply $ hexo new <name>. Edit the created file with Markdown or Hexo’s Helper. Save the edited file and proceed to deploy step for publishing the post. You could see all the changes on your local server before publishing them.

Step 8. Change Theme

I use an off-the-shelf theme hexo-theme-next. Surely you could write your own one or expore more at Hexo Themes.

Step 9. Deployment

$ hexo clean
$ hexo deploy

Step 10. ~ Done ~

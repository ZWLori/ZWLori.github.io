Blog Setup :Day

Posted on 2017-12-22

The blog is built based on hexo, which is a fast, simple & powerful blog framework (as what their main page shows).

Step 1. Check dependencies

  • Node.js
  • Git

Step 2. Install Hexo

$ npm install hexo-cli -g
Use $ hexo -v to check the version

Step 3. Init Hexo

$ hexo init <folder>
If you wish to use GitHub Pages, like what I did, then:

  • Create a project for your GitHub Pages
    Simply go to your github website and create a new project named as <your-github-id>.github.io.
  • $ hexo init <your-githubt-id>.github.io

Once initialized, here’s what the projet will look like:
.
|– _config.yml
|– package.json
|– scaffolds
|– source
| |– _drafts
| |– _posts
|– themes

_config.yml

Site configuration file that contains most settings.

package.json

Application data. The EJS, Stylus and Markdown renders are installed by default. If you want, you can uninstall them later.

scaffolds

Scaffold folder. When a new post is created, Hexo will build files based on the corresponding file in scffolds folder. For example, when you run the command $ hexo new photo "Fantastic Pic", Hexo will try to find photo.md in the scaffolds folder and build the post based on it.

source

Source folder. This is where you put your site’s content. Hexo ignores hidden files and files or folder whose names are prefixed with _(underscore), except the _post folder. Renderable files, such as HTML, Markdown, will be processed and put into the public folder, while other files will simply be copied.

theme

Theme folder. Hexo generates a static website by combining the site content with the theme. The default one would be ‘landscape’

Step 4. Set Blog Information

Fill in and modify the _config.yml file.
Please find the link here, which explaines all the setting specified in the file.

Step 5. Deployment Settings

A detailed discription for deployment could be found here.
I use Git for deplyment and here is what I did.

  • Install hexo-deployer-git via command $ npm install hexo-deplyer-git --save
  • Edit deploy settings in _config.yml.

Step 6. Run Test Server

Now a basic website is complete. Run a test server for your blog by $ hexo server. Visit localhost:4000 to see your pages.

Step 7. Make a New Post

Make a new post by simply $ hexo new <name>. Edit the created file with Markdown or Hexo’s Helper. Save the edited file and proceed to deploy step for publishing the post. You could see all the changes on your local server before publishing them.

Step 8. Change Theme

I use an off-the-shelf theme hexo-theme-next. Surely you could write your own one or expore more at Hexo Themes.

Step 9. Deployment

$ hexo clean
$ hexo deploy

Step 10. ~ Done ~
